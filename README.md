# Нежный Календарик (v 1.0.0)

Мобильное PWA-приложение для личного и приватного отслеживания лечения, симптомов, самочувствия и цикла.

## Что реализовано

- Mobile-first интерфейс с нижним таб-баром.
- Разделы: Сегодня, Календарь, Приемы, Симптомы, Еще (Цикл, Настройки, Экспорт/Импорт, О приложении).
- Полностью локальное хранение в `localStorage`.
- Онбординг с выбором: **«Заполнить по моей схеме»** или **«Начать с пустого»**.
- Предзаполненная схема лечения из задания (включая этап после месячных).
- Экспорт и импорт JSON с миграцией версии.
- PWA: `manifest.webmanifest`, service worker, установка на главный экран.
- Offline-first после первой загрузки.
- Базовая поддержка локальных уведомлений с graceful degradation.
- Медицинский дисклеймер в онбординге и настройках.

## Технологии

- React + TypeScript + Vite
- date-fns
- Vitest (тесты utility-логики)

## Очень подробно: как открыть сайт со стороннего телефона через GitHub Pages

Да — этот проект можно захостить через **GitHub Pages**, и он будет работать как сайт. PWA тоже будет работать, но есть нюансы (см. раздел «Ограничения»).

### Вариант A (рекомендуется): автоматический деплой через GitHub Actions

В репозитории уже добавлен workflow `.github/workflows/deploy-gh-pages.yml`.

#### Шаг 1. Создать репозиторий и запушить код

```bash
git init
git add .
git commit -m "feat: gentle calendar"
git branch -M main
git remote add origin https://github.com/<ваш_логин>/<имя_репозитория>.git
git push -u origin main
```

#### Шаг 2. Включить GitHub Pages в настройках

1. Откройте репозиторий на GitHub.
2. Перейдите в **Settings → Pages**.
3. В разделе **Build and deployment** выберите **Source: GitHub Actions**.
4. Сохраните.

#### Шаг 3. Дождаться первого деплоя

1. Откройте вкладку **Actions**.
2. Запустится workflow `Deploy to GitHub Pages` после push в `main`.
3. Дождитесь зеленого статуса `build` и `deploy`.
4. Ссылка на сайт будет вида:
   - `https://<ваш_логин>.github.io/<имя_репозитория>/`

#### Шаг 4. Открыть сайт на стороннем телефоне

1. Отправьте ссылку владельцу телефона (Telegram/WhatsApp/почта).
2. Откройте ссылку в браузере телефона.
3. Для установки как PWA: меню браузера → **Добавить на экран «Домой»**.

---

### Вариант B: ручной деплой из `dist` (если не хотите Actions)

1. Соберите проект:
   ```bash
   npm install
   npm run build
   ```
2. Опубликуйте содержимое `dist/` в ветку `gh-pages`.
3. В `Settings → Pages` выберите source: `Deploy from a branch`, branch: `gh-pages`, folder: `/ (root)`.

---

### Важные технические моменты для GitHub Pages (уже учтено)

- В `vite.config.ts` добавлен `base` через `VITE_BASE_PATH`.
- Workflow выставляет `VITE_BASE_PATH=/<repo-name>/` для корректной работы путей.
- `manifest.webmanifest` использует относительные `start_url`/`scope` и относительные пути к иконкам.
- Service Worker регистрируется через `import.meta.env.BASE_URL`.
- Добавлен `public/404.html` для мягкого fallback на SPA при прямых открытиях URL.

## Экспорт / Импорт данных

- В разделе **Настройки и данные** нажмите **Экспорт JSON**.
- Для импорта выберите файл `.json` через кнопку **Импорт JSON**.
- Поддерживается импорт более старых структур через слой миграции.

## Где хранятся данные

Только локально в браузере на этом устройстве (`localStorage`), без серверов, аккаунтов и облаков.

## Локальные уведомления: ограничения

- Реализованы только локальные browser notifications.
- Если разрешение не выдано, приложение продолжает работать без ошибок.
- На iOS уведомления PWA зависят от версии iOS и настроек устройства/браузера.
- Серверный push не используется.

## Ограничения при использовании GitHub Pages

1. **Данные не синхронизируются между телефонами автоматически**: каждый телефон хранит данные локально.
2. При очистке данных браузера история может удалиться.
3. Для переноса между телефонами используйте **Экспорт/Импорт JSON**.
4. После обновления версии сайта service worker может обновиться не мгновенно — иногда нужен перезапуск вкладки.

## Edge cases, которые учтены

- Изменение даты начала лечения и ручная корректировка расписания.
- Отметка месячных задним числом.
- Отмена ошибочного завершения месячных.
- Дедупликация событий при добавлении постменструального этапа.
- Импорт частичных/старых JSON через миграцию.
- Офлайн-открытие после первой загрузки через service worker.
