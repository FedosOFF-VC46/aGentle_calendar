# Сегодня + Календарь: функционал и логика работы

Этот документ описывает, как работают разделы **«Сегодня»** и **«Календарь»** после правок.

---

## 1) Раздел «Сегодня»

Файл интерфейса: `src/features/home/TodayScreen.tsx`.

### Что показывается на экране

1. Текущая дата (локализована на русский).
2. Карточка **«Сегодняшний план»**:
   - номер дня цикла;
   - список доз на текущую дату;
   - для каждой дозы: время, тип приема, препарат, дозировка и статус.
3. Карточка самочувствия (выбор настроения за день).

### Как формируется список приемов на сегодня

- Список `todayDoses` приходит из `useTreatmentPlan`.
- В хукe берутся дозы только с `dose.date === today` и сортируются по времени `currentTime`.
- Если доз на текущую дату нет, показывается текст «Сегодня все спокойно ✨».

### Статусы приема

Каждая доза может иметь один из статусов:

- `scheduled` — запланировано;
- `done` — принято;
- `skipped` — пропущено;
- `postponed` — отложено.

На карточке для дозы всегда отображается строка `Статус: ...`.

### Кнопки действий и их логика

#### Кнопка «Принято»

- Вызывает `updateDoseStatus(dose.id, 'done')`.
- После нажатия:
  - статус меняется на `done`;
  - кнопка визуально подсвечивается и блокируется (чтобы избежать повторного клика).

#### Кнопка «Пропустить»

- Вызывает `updateDoseStatus(dose.id, 'skipped')`.
- После нажатия:
  - статус меняется на `skipped`;
  - кнопка визуально активна и блокируется.

#### Кнопка «Отложить»

- Вызывает `updateDoseStatus(dose.id, 'postponed', '23:30')`.
- После нажатия:
  - статус меняется на `postponed`;
  - `currentTime` обновляется на `23:30`;
  - в карточке дополнительно показывается исходное время (`Было: plannedTime`), если оно изменилось.

#### Кнопка «Сбросить»

- Показывается только если статус уже изменен (`!= scheduled`).
- Вызывает `updateDoseStatus(dose.id, 'scheduled', dose.plannedTime)`.
- Возвращает дозу к исходному состоянию:
  - статус `scheduled`;
  - время снова становится исходным `plannedTime`.

### Где реально сохраняется изменение

- В `src/hooks/useTreatmentPlan.ts` функция `updateDoseStatus` вызывает `patch(...)`.
- `patch` меняет состояние приложения, которое хранится в localStorage через `useLocalStore`.
- Поэтому отметки «Принято/Пропустить/Отложить» сохраняются и не теряются при перезапуске страницы.

## 2) Раздел «Календарь»

Файл интерфейса: `src/features/calendar/CalendarScreen.tsx`.

### Основная идея

Календарь теперь не «просто точки», а рабочий обзор:

1. Есть переключение между месяцами (назад/вперед).
2. Есть шапка дней недели (Пн–Вс).
3. Есть легенда цветов (что означает каждая точка).
4. Можно выбрать конкретный день нажатием на ячейку.
5. Под календарем отображается сводка по выбранной дате.

### Как строится месяц

- Берется `visibleMonth` (текущий видимый месяц).
- Через date-fns строятся:
  - `monthStart` (1-е число месяца),
  - `monthEnd` (последнее число),
  - массив всех дней `days`.
- `leading` добавляет пустые ячейки до первого дня месяца, чтобы сетка совпадала с днями недели (понедельник — первый столбец).

### Что означают индикаторы в дне

Для каждой даты вычисляются признаки:

- есть приемы (`doses.length > 0`) — зеленая точка;
- все приемы выполнены (`done`) — темно-зеленая точка;
- есть запись симптомов — сиреневая точка;
- есть заметка — персиковая точка;
- дата входит в интервал цикла — розовая точка;
- есть задача лечения/плана — оранжевая точка.

### Выбор дня

- При клике на ячейку вызывается `setSelectedDate(date)`.
- Выбранный день получает выделение рамкой.
- Текущий день отдельно подсвечивается как `today`.
- Ниже выводится карточка-сводка:
  - количество приемов,
  - количество задач,
  - есть/нет симптомы,
  - есть/нет заметка,
  - идет/не идет цикл.

### Навигация по месяцам

- «← Месяц назад» = `subMonths(visibleMonth, 1)`.
- «Месяц вперед →» = `addMonths(visibleMonth, 1)`.

Таким образом календарь можно использовать как историю и как быстрый аудит соблюдения плана.

---

## 3) Связь «Сегодня» ↔ «Календарь»

- Когда в разделе **«Сегодня»** вы меняете статус дозы, изменяется `state.treatmentPlan.doses`.
- Раздел **«Календарь»** читает те же данные из `state`, поэтому:
  - появляются/обновляются индикаторы на соответствующей дате;
  - флаг «все приемы выполнены» пересчитывается автоматически.

Итог: это единый поток данных, а не два независимых экрана.
